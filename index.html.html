Cadena de Instrumentación Industrial
// FLUJO DE INFORMACIÓN: PROCESO → SENSOR → TRANSMISOR → CONTROLADOR → ACTUADOR → PROCESO
En un lazo de control industrial, la información fluye desde el proceso hacia el sistema de control y de vuelta al proceso. Cada elemento tiene una función específica y características normalizadas para garantizar la interoperabilidad. Las normas ISA-5.1, IEC 61511 y ISA-18.2 rigen la nomenclatura y el diseño de estos lazos.

▸ EJEMPLO REAL — INGENIO MANUELITA, VALLE DEL CAUCA
Control de nivel en evaporadores de jugo de caña:
El sensor de presión diferencial (DP cell) mide el nivel de jugo en el evaporador → el transmisor Rosemount 3051 convierte la señal a 4–20 mA → el DCS Honeywell Experion procesa la señal y ejecuta el algoritmo PID → envía señal 4–20 mA al posicionador → la válvula de control Fisher D4 (EFC) regula el flujo de entrada al evaporador. El protocolo HART permite leer el diagnóstico del instrumento sin interrumpir el lazo de control de 4–20 mA.
Punto clave de diseño: La selección del protocolo de comunicación depende de la velocidad de actualización requerida, la distancia, la cantidad de instrumentos en el bus y los requisitos de seguridad funcional (SIL). No existe un protocolo universal para todas las aplicaciones.
CONCEPTO CLAVE
Lazo de Control Cerrado
El lazo cerrado compara continuamente el valor medido (PV) con el set-point (SP). El error e = SP − PV es procesado por el controlador para generar la señal de salida (MV) que actúa sobre el proceso a través del elemento final de control.
CONCEPTO CLAVE
Variable de Proceso (PV)
Magnitud física que se desea controlar: temperatura (°C/K), presión (kPa/bar), caudal (m³/h), nivel (%), composición (% v/v), pH, conductividad. El sensor convierte la PV en una señal eléctrica procesable.
NORMA ISA
Tag de Instrumento
Según ISA-5.1: FIC-101 = Flow (F) · Indicating (I) · Controller (C) · Loop 101. El tag identifica inequívocamente cada instrumento en los P&ID de la planta, facilitando el mantenimiento y la documentación.
Elementos del Lazo de Control
// SENSOR · TRANSDUCTOR · TRANSMISOR · CONTROLADOR · ACTUADOR · EFC
🔬
ELEMENTO 1 — SENSOR
Sensor
Elemento en contacto directo con el proceso que percibe la variable física y genera una respuesta proporcional (generalmente una variación de resistencia, capacitancia, voltaje o frecuencia). No necesariamente genera señal normalizada.
  • Thermocouple (TC): Efecto Seebeck → μV proporcional a ΔT. Tipos J, K, T, E, R, S, B.
  • RTD Pt-100: Variación de resistencia con temperatura (0.385 Ω/°C).
  • Sensor capacitivo de nivel: Cambio de capacitancia C = εA/d.
  • Celda de presión diferencial: Diafragma → deformación → señal.
  • Electrodo de pH: FEM proporcional a log[H⁺].
EXACTITUD REPETIBILIDAD RANGO RESOLUCIÓN
🔄
ELEMENTO 2 — TRANSDUCTOR
Transductor
Dispositivo que convierte energía de una forma a otra. En instrumentación industrial, convierte la señal del sensor a una forma más adecuada para transmisión. El transductor puede ser parte del sensor o un elemento separado.
  • I/P (Current-to-Pressure): Convierte 4–20 mA a 3–15 psi neumático.
  • P/I (Pressure-to-Current): Convierte presión neumática a señal eléctrica.
  • Piezoeléctrico: Fuerza mecánica → carga eléctrica (cuarzo).
  • Strain gauge: Deformación mecánica → variación de resistencia.
  • LVDT: Desplazamiento lineal → voltaje AC.
LINEALIDAD HISTÉRESIS SENSIBILIDAD
📡
ELEMENTO 3 — TRANSMISOR
Transmisor
Convierte la señal del sensor/transductor a una señal normalizada estándar de industria (4–20 mA, 1–5 V, HART, Profibus PA, Foundation Fieldbus) para transmisión a larga distancia al sistema de control. Incluye acondicionamiento de señal.
  • Transmisores inteligentes: Microprocesador interno → auto-calibración, diagnóstico, HART.
  • Loop-powered (2 hilos): Se alimenta del mismo lazo 4–20 mA.
  • Span y Zero ajustables: Reconfiguración local o remota vía HART.
  • Ejemplo: Yokogawa EJX110A para presión diferencial.
  • Protección: ATEX / IECEx para zonas clasificadas.

ELEMENTO 4 — CONTROLADOR
Controlador
Recibe la señal de la variable de proceso (PV), la compara con el set-point (SP) y calcula la acción de control (MV) para minimizar el error. Implementa los algoritmos de control (PID, cascada, feedforward, IMC).
  • PLC (Programmable Logic Controller): Control discreto y continuo. Siemens S7, Allen-Bradley.
  • DCS (Distributed Control System): Control distribuido para plantas grandes. Emerson DeltaV, ABB 800xA.
  • SCADA: Supervisión y adquisición de datos a nivel de planta.
  • Control PID: Proporcional + Integral + Derivativo.
  • Safety PLC (SIS): IEC 61511 para funciones de seguridad SIL 1-3.
PID CASCADE FEEDFORWARD IMC
⚡
ELEMENTO 5 — ACTUADOR
Actuador
Convierte la señal eléctrica del controlador en acción mecánica sobre el elemento final de control. El actuador recibe la señal de mando (4–20 mA) y genera la fuerza/torque necesario.
  • Actuador neumático: Señal neumática 3–15 psi → fuerza sobre vástago de válvula. Requiere I/P transductor.
  • Actuador eléctrico: Motor eléctrico con encoder de posición. Señal 4–20 mA directa.
  • Actuador hidráulico: Alta fuerza, válvulas grandes. Servo-válvula hidráulica.
  • Posicionador: Dispositivo de retroalimentación que asegura posición exacta del actuador.
NEUMÁTICO ELÉCTRICO HIDRÁULICO
🚪
ELEMENTO 6 — EFC
Elemento Final de Control
Es el elemento que físicamente modifica la variable manipulada del proceso (flujo, temperatura, composición). Actúa directamente sobre el fluido o la energía del proceso.
  • Válvula de control: EFC más común. Características: lineal, equal %, quick-opening.
  • Variador de frecuencia (VFD): Modifica velocidad de bomba/compresor.
  • Damper (compuerta): Control de flujo de gases en ductos.
  • Resistencia eléctrica: Control de temperatura por calor Joule.
  • Criterio Cv: Capacidad de flujo de válvula (gal/min @ ΔP=1 psi).

Características Metrológicas de los Instrumentos
📐 Rango, Span y Campo de Medida
+
Rango (Range): Conjunto de valores de la variable comprendidos entre los límites superior e inferior de la escala del instrumento. Ej: −50 a 150°C.

Span (Alcance): Diferencia algebraica entre los límites superior e inferior del rango. Span = LimSup − LimInf. Ej: Span = 150 − (−50) = 200°C.

Campo de medida elevado (Elevated zero): Cuando el cero del instrumento se ha desplazado hacia arriba del punto de referencia. Ej: instrumento calibrado de 50 a 150°C (rango elevado).

Campo de medida suprimido (Suppressed zero): Cuando el cero está por debajo del límite inferior de la escala. Importante en transmisores de presión diferencial para medición de nivel.
🎯 Exactitud, Precisión y Resolución
+
Exactitud (Accuracy): Grado de concordancia entre el valor medido y el valor verdadero. Se expresa como % del span (±0.1% span) o % del valor leído.

Precisión (Precision): Dispersión de las medidas repetidas alrededor de la media. Un instrumento puede ser preciso sin ser exacto (sesgo sistemático).

Resolución: Variación mínima detectable de la variable de entrada. En transmisores inteligentes con conversores A/D de 16 bits: resolución = Span/65535.

Repetibilidad: Capacidad del instrumento de reproducir lecturas idénticas bajo condiciones iguales. Crítico en lazos de control tight.

Ejemplo industrial: Transmisor Endress+Hauser Promag 55S para flujo de ácido sulfúrico: exactitud ±0.2% del valor medido, repetibilidad ±0.1%.
⏱️ Dinámica del Instrumento: Tiempo de Respuesta y Lag
+
Tiempo de respuesta (t₉₀): Tiempo para que el instrumento alcance el 90% del valor final ante un escalón. Crítico cuando la dinámica del instrumento compite con la del proceso.

Tiempo muerto (Dead time θ): Retardo puro desde que ocurre el cambio hasta que el instrumento comienza a responder. Termopozo largo → mayor θ.

Constante de tiempo τ: Tiempo para alcanzar el 63.2% del valor final. Para un instrumento de 1er orden: G(s) = K/(τs+1).

Regla práctica: Para control efectivo, τ_instrumento < τ_proceso/10. Si el instrumento es 10 veces más lento que el proceso, se convierte en el elemento dominante del lazo.

Termopozos: Añaden dinámica adicional al lazo de temperatura. Se modela como sistema de 2do orden o sistema con tiempo muerto equivalente.
📊 Histéresis, Zona Muerta y Deriva
+
Histéresis: Diferencia máxima entre las lecturas ascendentes y descendentes del instrumento para el mismo valor de entrada. Causa oscilaciones en lazos de control con alta ganancia. Se expresa como % del span.

Zona muerta (Dead band): Rango de variaciones de la entrada alrededor de un punto dado para el cual no se produce cambio en la salida. En válvulas de control: stiction (fricción estática) genera zona muerta que causa cycling.

Deriva (Drift): Variación lenta y gradual de la lectura del instrumento bajo condiciones constantes de la variable. Causa: cambios de temperatura ambiente, envejecimiento de componentes.

Solución práctica: Programa de calibración periódica según ISA-18.2. Transmisores inteligentes con diagnóstico de deriva integrado (Emerson AMS Suite).
Señales & Estándares Industriales

〜
SEÑAL ANALÓGICA ESTÁNDAR
4–20 mA (Loop Current)
Estándar dominante en instrumentación de proceso industrial desde los años 60. La corriente (no el voltaje) garantiza inmunidad al ruido en distancias largas. El valor de 4 mA representa el 0% de la escala (no cero), permitiendo detectar cables cortados (0 mA = falla).
  • Rango vivo: 4 mA (0%) → 20 mA (100%)
  • Falla de cable: 0 mA → alarma de bajo nivel
  • Saturación: 3.8 mA / 20.5 mA (bajo/alto alarm)
  • Distancia: Hasta 1 km con resistencia de loop ≤ 600 Ω
  • Alimentación: 24 VDC loop-powered (2 hilos)
📶
PROTOCOLO HÍBRIDO
HART (Highway Addressable Remote Transducer)
Superpone una señal digital FSK (±0.5 mA, 1200/2200 Hz) sobre el lazo 4–20 mA. No interfiere con la señal analógica. Permite comunicación bidireccional con el instrumento sin interrumpir el control. Estándar de facto para mantenimiento y diagnóstico.
  • Velocidad: 1200 bps (lento, pero suficiente para configuración)
  • Variables de proceso: Hasta 4 PV por dispositivo
  • Modo multidrop: Hasta 15 dispositivos en un bus (sin señal analógica)
  • WirelessHART: ISA100.11a → HART sin cableado (mesh network)
  • Uso típico: Calibración remota, diagnóstico, configuración de span/zero
💡
SEÑAL DISCRETA
NAMUR & Señales On/Off
Para instrumentos de seguridad en áreas clasificadas. La norma NAMUR (EN 60947-5-6) define señales de corriente pequeña (1–8 mA) para sensores de proximidad, finales de carrera y válvulas. Permite distinguir entre "abierto", "cerrado" y "falla de cable".
  • Activo (ON): 6–8 mA
  • Inactivo (OFF): 1–1.8 mA
  • Cortocircuito: > 8 mA → falla detectada
  • Cable abierto: < 1 mA → falla detectada
  • Aplicación: Detectores de llama, sensores de posición en válvulas
🌐
SEÑAL DIGITAL
Foundation Fieldbus H1 & PA
Bus de campo digital completamente digital para el nivel de campo. Elimina la señal analógica 4–20 mA. Los dispositivos son inteligentes con bloques funcionales que ejecutan control localmente. Soporta alimentación y comunicación en el mismo par de hilos.
  • FF H1: 31.25 kbps, hasta 32 dispositivos por segmento
  • FF HSE: 100 Mbps, Ethernet industrial en sala de control
  • Profibus PA: 31.25 kbps, protocolo alemán (Siemens), compatible H1
  • Ventaja: Control en el campo → menor dependencia del DCS
  • Diagnóstico avanzado: Señales de diagnóstico estandarizadas
▸ CONVERSIÓN DE SEÑAL: CÁLCULO PRÁCTICO
Ejemplo: Transmisor de presión, rango 0–600 kPa, señal 4–20 mA.
Si el transmisor envía 12 mA, ¿cuál es la presión medida?

Fórmula: PV = PV_min + [(I − 4 mA) / 16 mA] × (PV_max − PV_min)
PV = 0 + [(12 − 4) / 16] × 600 = (8/16) × 600 = 300 kPa

Para la señal HART superpuesta: Las frecuencias de 1200 Hz (lógico "1") y 2200 Hz (lógico "0") no afectan la medición del multímetro de corriente porque el promedio de la señal FSK es cero.
Protocolos de Comunicación Industrial
// NIVEL DE CAMPO · NIVEL DE CONTROL · NIVEL DE PLANTA
La arquitectura de comunicación industrial sigue el modelo de pirámide de automatización CIM (Computer Integrated Manufacturing): nivel de campo → nivel de control → nivel de planta → nivel corporativo. Cada nivel tiene requisitos de velocidad, determinismo y capacidad de datos diferentes.
Nivel de Campo (Field Level)
🔗 HART — Highway Addressable Remote Transducer
+
Año: 1986 (Rosemount Inc.) → Ahora administrado por FieldComm Group.
Física: FSK sobre 4–20 mA. Modulación Bell 202.
Velocidad: 1200 bps.
Topología: Punto a punto (con señal analógica) o multidrop (sin analógica).
Protocolo: Maestro-esclavo. Soporta 2 maestros (primario y secundario).
Uso industrial: >80% de los instrumentos de campo actuales tienen HART. Es el protocolo de mantenimiento estándar en la industria. Herramientas: AMS, FieldCare, PDM.
WirelessHART (IEC 62591): Red mesh 2.4 GHz, seguridad AES-128, latencia ajustable.
🔗 Foundation Fieldbus (FF H1)
+
Año: 1994. Estándar IEC 61158.
Física: Par trenzado, 31.25 kbps. Alimentación y datos en el mismo bus.
Topología: Bus, árbol, cascada (con repetidores). Hasta 32 dispositivos / segmento (12 con certificación Ex).
Protocolo: Basado en bloques funcionales (FB): AI, AO, PID, Cascade, etc. Control se puede ejecutar en el dispositivo de campo.
Ventaja clave: Comunicación determinista con LAS (Link Active Scheduler). Latencia determinista garantizada.
Uso en Colombia: Planta de fertilizantes Abocol (Cartagena), Ecopetrol Barrancabermeja.
Limitación: Desplazado por HART+AMS y Ethernet industrial en instalaciones nuevas.
🔗 Profibus PA / DP
+
Año: 1989 (Siemens). Estándar IEC 61158 / EN 50170.
Profibus DP (Decentralized Periphery): RS-485, hasta 12 Mbps, para I/O distribuido (conexión PLC ↔ remote I/O). Alta velocidad, determinismo.
Profibus PA (Process Automation): 31.25 kbps, IEC 61158-2 (idéntico a FF H1 físicamente). Para instrumentación de proceso en áreas clasificadas.
DP/PA coupler: Dispositivo que une los dos buses (conversión de velocidad).
Uso masivo: Plantas Siemens, BASF, industria química alemana/europea. Dominante en Europa, competidor directo de FF en procesos continuos.
Diagnóstico: Profibus Profile 3.02 → diagnóstico extendido de dispositivos.
🔗 Modbus RTU / TCP-IP
+
Año: 1979 (Modicon). Protocolo abierto más antiguo y simple.
Modbus RTU: RS-232 / RS-485, hasta 115 kbps. Protocolo maestro-esclavo. Registros de 16 bits (Holding Registers, Input Registers, Coils, Discrete Inputs).
Modbus TCP/IP: Modbus sobre Ethernet (puerto 502). Sin cambios en la estructura del mensaje, solo encapsulado en TCP.
Uso masivo: PLCs, drives, medidores de energía (Power meters), analizadores de gas, UPS industriales. Omnipresente por su simplicidad.
Limitación: No determinista en TCP. Sin seguridad nativa. Sin mecanismos de diagnóstico avanzado.
Ejemplo: Medidores de energía Schneider PM8000 en subestaciones de plantas.
🔗 OPC UA (OLE for Process Control Unified Architecture)
+
Año: 2008 (OPC Foundation). Estándar IEC 62541.
Descripción: Arquitectura orientada a servicios (SOA) para intercambio de datos industriales. Independiente de plataforma, SO, lenguaje de programación.
Características clave: Seguridad integrada (certificados X.509, encriptación AES-256), modelo de información semántico (nodos con metadatos), comunicación pub/sub y cliente/servidor.
Rol en Industria 4.0: Es el "lenguaje universal" para integración PLC ↔ MES ↔ ERP ↔ Cloud. Todos los fabricantes (Siemens, ABB, Emerson, Rockwell) lo soportan.
OPC UA over MQTT: Para IIoT y edge computing. Latencia baja, escalable a millones de dispositivos.
Uso en Colombia: Integración de SCADA con SAP en plantas de Ecopetrol, Bavaria, Postobón.

100 Mbps / 1 Gbps. Determinismo logrado por mecanismos adicionales sobre Ethernet estándar IEEE 802.3. PROFINET IO para integración PLC Siemens. EtherNet/IP para ecosistema Rockwell/Allen-Bradley.

Modelo OSI Industrial
// CAPAS DE COMUNICACIÓN — DE LA FÍSICA AL USUARIO
El modelo OSI (Open Systems Interconnection, ISO/IEC 7498) define 7 capas de comunicación. Los protocolos industriales implementan un subconjunto de estas capas. Por ejemplo, Modbus RTU solo implementa las capas 1, 2 y 7 (sin capas 3–6). Foundation Fieldbus utiliza las capas 1, 2 y 7, con el "User Layer" adicional para bloques funcionales.

FÍSICA
▸ ANÁLISIS DE CAPAS POR PROTOCOLO
Modbus RTU: Capa 1 (RS-485) + Capa 2 (trama Modbus, CRC-16) + Capa 7 (Function Codes). Capas 3-6: vacías.

HART: Capa 1 (4–20 mA FSK) + Capa 2 (trama HART, CRC) + Capa 7 (HART Commands, DDL). Capas 3-6: vacías.

Foundation Fieldbus H1: Capa 1 (IEC 61158-2) + Capa 2 (DLL con LAS scheduler) + Capa 7 (FMS/FAS) + User Layer (bloques funcionales). Capas 3-6: vacías.

OPC UA sobre TCP/IP: Todas las 7 capas + seguridad adicional (TLS, certificados X.509). Protocolo más completo.

Aplicación: Hornos de crudo, torres de destilación, sistemas de gas natural, compresores de alta presión.

Arquitectura: Instrumentos FF H1 / HART → Emerson DeltaV DCS → OPC UA → SAP PM para mantenimiento predictivo → Emerson AMS Device Manager para diagnóstico de campo.
  • Temperatura: Termopares tipo K + transmisor Rosemount 848T (FF multicanal)
  • Presión: Yokogawa EJX Series (HART) en servicios criogénicos
  • Flujo másico: Micromotion Elite (Coriolis) → Foundation Fieldbus
  • Safety: SIS Triconex con protocolo propietario → IEC 61511 SIL 2/3
  • Gas cromatógrafo: Siemens MAXUM → Modbus TCP → DCS

SECTOR AGUA — CALI
PTAR Cañaveralejo / EMCALI
Aplicación: Monitoreo de calidad de agua tratada, control de dosificación de cloro, caudales de entrada/salida.

Arquitectura: Sensores en campo → RTU (Remote Terminal Unit) → Comunicación Modbus RTU/TCP → SCADA Wonderware → Red GPRS/4G para puntos remotos.
  • pH, OD, turbidez: Sondas Hach → transmisor SC1500 → Modbus RTU → RTU → SCADA
  • Caudal: Caudalímetro electromagnético Siemens MAG5100 → Profibus DP → PLC S7-300
  • Nivel en tanques: Radar Vega → 4–20 mA HART → RTU → SCADA
  • Telemetría: Protocolo DNP3 para comunicación con centros de control remotos
  • Dosificación de cloro: Control en cascada, señal de pH como variable primaria

Evolución Histórica: Señales de Campo
1940–1950s
Era Neumática
Señal neumática 3–15 psi (0.2–1 bar). Instrumentos puramente mecánicos. Aún presente en válvulas actuadas neumáticamente como estándar de facto.
1960–1970s
Señal 4–20 mA
ISA 50.1: señal eléctrica analógica estándar. Reemplaza paulatinamente la señal neumática para transmisión. Inmunidad al ruido, cable fino, detección de rotura.
1986
HART Protocol
Rosemount superpone comunicación digital sobre 4–20 mA. Primera vez que es posible configurar un instrumento desde la sala de control sin interrumpir el lazo de control.
1990s
Buses de Campo (Fieldbus Wars)
IEC y ISA intentan estandarizar: Foundation Fieldbus vs Profibus vs DeviceNet vs Interbus vs... Resultado: múltiples estándares coexistentes (IEC 61158 incluye 8 tipos).
2000s
Ethernet Industrial
PROFINET, EtherNet/IP, EtherCAT llevan Ethernet determinista al nivel de control. OPC DA/UA para integración vertical. El bus de campo pierde terreno en plantas nuevas.
2010s–Hoy
IIoT, Cloud & Industria 4.0
OPC UA como lingua franca. WirelessHART e ISA100. Digital Twin, edge computing, ML para mantenimiento predictivo. APL (Advanced Physical Layer): Ethernet hasta el sensor.
