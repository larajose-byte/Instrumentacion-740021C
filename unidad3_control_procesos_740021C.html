Lazo Abierto vs Lazo Cerrado
// REALIMENTACIÓN NEGATIVA · GANANCIA · VELOCIDAD · ERROR · PERTURBACIONES
La diferencia fundamental entre lazo abierto y cerrado es la realimentación: en lazo cerrado, la salida del proceso se mide continuamente y se compara con el set-point para calcular el error y generar la acción correctiva. Esta realimentación negativa es lo que da al lazo cerrado sus propiedades superiores de rechazo de perturbaciones y reducción de error — pero también introduce el riesgo de inestabilidad.
LAZO ABIERTO
✗ Sin realimentación de la variable controlada
✗ Error de offset permanente si hay perturbaciones
✗ Sensible a cambios del proceso (Kp, τ, θ)
✓ Siempre estable (no puede volverse inestable)
✓ Simple: no requiere sintonización de Gc
✓ Útil para procesos muy conocidos y estables
Ejemplo: precalentador con válvula en posición fija
vs
LAZO CERRADO
✅ Realimentación negativa corrige error continuamente
✅ Elimina offset (con acción I) ante perturbaciones
✅ Robusto ante variaciones del proceso
✅ Rechaza perturbaciones de carga automáticamente
⚠ Puede volverse inestable si Gc mal sintonizado
⚠ Requiere transmisor, controlador y sintonización
Ejemplo: lazo TIC de temperatura en reactor exotérmico

Acciones de Control — P · I · D
// PROPORCIONAL · INTEGRAL · DERIVATIVA · PI · PD · PID IDEAL Y REAL
La ley PID combina tres acciones sobre el error e(t)=SP−PV. P corrige proporcionalmente al error presente, I elimina el error acumulado histórico y D anticipa el error futuro. Experimente con el simulador interactivo abajo para visualizar el impacto de cada parámetro.
📐 Ley de Control PID — Formas Paralela, Serie e Ideal
Forma paralela (estándar ISA): u(t) = Kc·[e(t) + (1/τI)·∫e dt + τD·de/dt] + u_bias
Función de transferencia: Gc(s) = Kc·(τI·τD·s² + τI·s + 1)/(τI·s)
Forma serie (interactuante): Gc(s) = Kc·(1+1/(τI·s))·(1+τD·s)
PID real (con filtro derivativo N=5–20): Gc(s) = Kc·[1 + 1/(τI·s) + τD·s/(τD·s/N + 1)]

Impacto de Cada Acción
Acción P — Proporcional
Salida proporcional al error actual. Base de todo controlador. ↑ Kc → más rápido, menos margen. Produce offset permanente ante perturbaciones sostenidas. P
u_P = Kc·e(t) → e_ss = 1/(1+Kp·Kc) ≠ 0
Acción I — Integral
Elimina offset acumulando el error. ↓ τI → más agresiva. Peligro: windup ante saturación del actuador. I
u_I = (Kc/τI)·∫e dt → e_ss = 0 (tipo 1)
Acción D — Derivativa
Anticipa el error usando de/dt. Agrega amortiguamiento → reduce OS. ¡Amplifica ruido de alta frecuencia! Siempre usar con filtro N. D
u_D = Kc·τD·de/dt → Gc_D(s)=Kc·τD·s/(τD·s/N+1)
🎬 Videos — Acción de Control
Anti-windup: Cuando la válvula satura (100% o 0%), la acción I sigue acumulando error — al salir de saturación, el integrador desenrolla y produce sobreimpulso severo. Los DCS industriales siempre implementan: back-calculation, conditional integration o clamping. Sin anti-windup un arranque puede ser peligroso.
❓ Con ruido de ±0.3°C en el termopar, ¿qué controlador es más adecuado?
A. PID con τD grande para anticipar cambios
B. PI con filtro de ruido en PV (recomendado)
C. PD porque es más rápido que PI
D. Solo P porque el ruido no importa
✅ La acción D amplifica ruido de alta frecuencia exponencialmente (de_D/dt). Se prefiere PI que elimina offset sin amplificar ruido. Si se necesita D, siempre con filtro (N=5–10).

Índices Integrales de Desempeño
ISE — Integral of Squared Error
Penaliza fuertemente los errores grandes (cuadráticos). Optimiza la sintonía más agresiva: respuesta rápida pero mayor sobreimpulso. Ideal cuando los errores grandes son inaceptables.
ISE = ∫₀^∞ e²(t) dt [mínimo → óptimo]
IAE — Integral of Absolute Error
Penaliza igual todos los errores. Balance entre rapidez y amortiguamiento. Más robusto que ISE. El criterio más usado en la industria química para sintonización práctica.
IAE = ∫₀^∞ |e(t)| dt [mínimo → óptimo]
ITAE — Integral Time Absolute Error
Penaliza más los errores tardíos que los iniciales (peso t). Produce la sintonía más amortiguada — mínimo sobreimpulso — a costa de respuesta inicial más lenta. Recomendado para procesos de seguridad.
ITAE = ∫₀^∞ t·|e(t)| dt [mínimo → óptimo]
ITSE — Integral Time Squared Error
Combina el peso temporal con penalización cuadrática. Balance entre ISE e ITAE. Menos usado que IAE/ITAE pero útil para procesos donde tanto la magnitud como la persistencia del error son críticas.
ITSE = ∫₀^∞ t·e²(t) dt [mínimo → óptimo]

Diseño PID por Asignación de Polos
// POLINOMIO DESEADO · IGUALACIÓN DE COEFICIENTES · ΖETA Y OMEGA_N DESEADOS
La asignación de polos (pole placement) es un método analítico que diseña el controlador PID de forma que los polos del lazo cerrado queden exactamente en las ubicaciones deseadas en el plano s. Si el ingeniero desea ζ=0.707 y ωn=2 rad/s, calcula Kc, τI, τD algebraicamente igualando el polinomio característico obtenido con el polinomio deseado. Garantiza las especificaciones exactas — a diferencia de los métodos empíricos que son aproximados.
📐 Procedimiento de Asignación de Polos para Proceso FOPDT-Aproximado
Proceso (1er orden sin tiempo muerto — aproximado): G(s) = Kp/(τs+1)
Controlador PID paralelo: Gc(s) = Kc·(1 + 1/(τI·s) + τD·s)
Polinomio característico del LC: 1 + Gc(s)·G(s) = 0
→ τ·τI·s³ + [τI + Kp·Kc·τD·τI]·s² + [1 + Kp·Kc·τI]·s + Kp·Kc = 0
Polinomio deseado (polos ζ, ωn + polo rápido en −p, p≥5ωn):
P_des(s) = (s² + 2ζωn·s + ωn²)·(s + p)
= s³ + (2ζωn+p)·s² + (ωn²+2ζωn·p)·s + ωn²·p
Igualar coeficientes → 3 ecuaciones → 3 incógnitas (Kc, τI, τD):
Kp·Kc = ωn²·p·τ → Kc = ωn²·p·τ/Kp τI = (1 + Kp·Kc·τI) / (ωn²+2ζωn·p) → resolver el sistema τD = ...

P_des(s) = (s²+2×0.707×1×s+1²)·(s+8) = (s²+1.414s+1)·(s+8)
= s³ + 9.414s² + 12.312s + 8
Igualando coeficientes (dividiendo por coef. de s³ = 8/τI):
De s⁰: 1.5·Kc = 8·τI/τ × ... → 1.5·Kc/τI = 8/8 → Kc/τI = 1/1.5 = 0.667
De s⁰ igualado: Kp·Kc/τI = ωn²·p = 1×8 = 8 → 1.5·Kc/τI = 8 → Kc/τI = 5.33

📐 Cómo elegir ζ, ωn y el polo rápido p para el diseño
+
Selección de ζ (coeficiente de amortiguamiento):
• ζ = 0.707 (1/√2): estándar industrial — OS≈4.3%, balance óptimo velocidad/amortiguamiento
• ζ = 0.5: permite 16% OS — más rápido, aceptable si OS no es crítico
• ζ = 1.0: sin sobreimpulso — para reactores exotérmicos o procesos de seguridad

Selección de ωn (frecuencia natural deseada):
• ωn = 1/τ: lazo cerrado igual de rápido que el proceso → buena partida
• ωn = 3/τ: lazo 3× más rápido — verificar que θ no limite (θ<1/ωn)
• Restricción: ωn ≤ 1/(3θ) para garantizar estabilidad con tiempo muerto

Selección del polo rápido p:
• p ≥ 5·ωn: polo no dominante — se "cancela" en la respuesta
• p = 10·ωn: muy rápido → parámetros PID más exigentes (τD muy pequeño)
• En la práctica: p = 5–8·ωn es el rango más usado

Limitación: Para procesos con tiempo muerto significativo (θ/τ>0.3), la aproximación de primer orden linealizado del tiempo muerto (1−θs) puede introducir error. En esos casos, usar IMC o verificar con Bode que Pm≥30°.

Controladores PID Digitales
// DISCRETIZACIÓN · TUSTIN (BILINEAL) · ZOH · MAPEO POLO-CERO · PERÍODO DE MUESTREO
En la práctica industrial, los controladores PID se implementan en microprocesadores, PLCs y DCS que operan en tiempo discreto: muestrean la señal de PV cada T segundos y calculan una nueva señal MV. El controlador continuo Gc(s) debe convertirse a su equivalente discreto Gc(z) mediante un método de discretización adecuado. La elección del período de muestreo T es crítica para preservar el comportamiento del controlador continuo.
Métodos de Discretización
Euler Hacia Adelante (Forward)
Aproximación de la derivada como diferencia hacia adelante. Simple pero puede ser inestable incluso para sistemas estables continuos. Rara vez usada en control.
s → (z−1)/T → integrador: 1/s → T/(z−1)
Euler Hacia Atrás (Backward)
Más estable que forward. Mapea el semiplano izquierdo de s en el interior del círculo unitario de z. Introduce pequeño error de fase. Usada en PLCs industriales simples.
s → (z−1)/(T·z) → integrador: T·z/(z−1)
Tustin / Bilineal (Recomendado)
Mapeo exacto del eje jω → círculo unitario. Preserva mejor la respuesta en frecuencia. Introduce distorsión de frecuencia ("warping") corregible con prewarping. Estándar industrial.
s → 2(z−1)/[T(z+1)] → bilineal exacto
Mapeo Polo-Cero (ZOH)
Mapea cada polo y cero de Gc(s) individualmente: pᵢ → e^(pᵢT) en el plano z. Preserva exactamente la respuesta al escalón. Útil para T grande relativo a τ del controlador.
pᵢ → e^(pᵢ·T) y zⱼ → e^(zⱼ·T) (mapeo exacto)
PID Discreto — Tustin (Forma Posicional y Velocidad)
📐 Algoritmo PID Discreto — Derivación con Tustin
Gc(s) = Kc·[1 + 1/(τI·s) + τD·s] → sustiuir s = 2(z−1)/[T(z+1)]
Término proporcional: uP[k] = Kc·e[k]
Término integral (Tustin): uI[k] = uI[k−1] + (Kc·T)/(2·τI)·(e[k] + e[k−1])
Término derivativo (Tustin): uD[k] = (2τD−T)/(2τD+T)·uD[k−1] + 2Kc·τD/(2τD+T)·(e[k]−e[k−1])
Salida total: u[k] = uP[k] + uI[k] + uD[k] + u_bias
Forma de velocidad: Δu[k] = u[k] − u[k−1] → evita reset windup automáticamente y facilita el paso manual↔automático

Arquitecturas de Control
// SERIE · PARALELO · ESTRUCTURAS AVANZADAS · 2-DOF · CONTROL RATIO
Las arquitecturas de control determinan cómo se conectan los bloques del lazo: el controlador, el proceso, la realimentación y las perturbaciones. El lazo PID simple (1 grado de libertad) es el punto de partida, pero muchos problemas industriales requieren estructuras más sofisticadas que desacoplen el seguimiento del SP del rechazo de perturbaciones, o que manejen procesos de gran tiempo muerto o con perturbaciones medibles.
Control Serie vs Paralelo
CONTROL SERIE (2-DOF)
Dos Grados de Libertad
El controlador tiene dos entradas: SP directamente y el error. El filtro de referencia Fr(s) modifica la respuesta al cambio de SP sin afectar el rechazo de perturbaciones. Permite optimizar seguimiento y perturbación de forma independiente.
U(s) = Gc(s)·[Fr(s)·R(s) − Y(s)]
Fr(s) = 1/(τf·s+1) típico
CONTROL PARALELO
Un Grado de Libertad
Estructura estándar: el controlador actúa sobre el error E=R−Y. Los parámetros Kc, τI, τD deben balancear simultáneamente el seguimiento de SP y el rechazo de perturbaciones — lo que a veces genera compromiso entre los dos objetivos.
U(s) = Gc(s)·E(s) = Gc(s)·[R(s) − Y(s)]
FIG. 4 — Comparativa 1-DOF vs 2-DOF. En 2-DOF, el filtro Fr(s) acondiciona el cambio de SP antes del sumador, permitiendo que Gc se optimice únicamente para rechazar perturbaciones. Fr(s)=1/(τf·s+1) con τf≈τD típicamente.
📐 Control de Relación (Ratio Control)
+
El control de relación mantiene la proporción entre dos caudales: F₂/F₁ = R_deseado (ratio). Común en: proporción aire/combustible en hornos, relación de alimentación en reactores, mezcla de corrientes.

Estructura:
1. F₁ se mide (variable "wild" — no controlada)
2. SP₂ = R × F₁ (set-point calculado en tiempo real para F₂)
3. Controlador de flujo FIC-2 mantiene F₂ = SP₂

Con corrección de ratio: Un analizador en línea mide la composición resultante → ajusta R en tiempo real → forma un lazo cascada con el control de ratio.

En MATLAB: Se implementa como dos lazos: un bloque multiplicador R×F₁ genera el SP del segundo lazo FIC-2.

Predictor de Smith
// COMPENSACIÓN DE TIEMPO MUERTO · MODELO INTERNO · θ GRANDE · θ/τ ≥ 0.3
¿Cuándo usar el Predictor de Smith? Cuando θ/τ ≥ 0.3. Para procesos con gran tiempo muerto (columnas, hornos, procesos con transporte largo), el PID convencional tiene márgenes reducidos. El Predictor de Smith "saca" el tiempo muerto del lazo interno, permitiendo diseñar el PID como si no hubiera θ.

T(s) = Gc·Gm·e^(−θs)/[1+Gc·Gm] ← θ solo en numerador → no afecta estabilidad del lazo interno

Control Cascada
// LAZO MAESTRO · LAZO ESCLAVO · RECHAZO ANTICIPADO · DISEÑO PASO A PASO
El control cascada usa dos controladores en serie: el lazo esclavo (interno, más rápido) controla una variable secundaria, y el lazo maestro (externo, más lento) controla la variable principal y genera el SP del esclavo. La clave: el esclavo rechaza perturbaciones internas antes de que afecten la variable principal.
Reglas de Diseño
  • Regla de velocidad: El lazo esclavo debe ser al menos 3–5× más rápido que el maestro. Si τ₂ > τ₁/3, la cascada no aporta beneficio.
  • Sintonizar primero el esclavo: Con lazo maestro en manual, sintonizar Gc₂ hasta respuesta rápida sin oscilación. Típicamente P o PI con ganancia alta.
  • Sintonizar el maestro: Con esclavo en automático, el maestro ve G₁·LC₂ como proceso efectivo. Usar Z-N, IMC o CSD sobre este modelo combinado.
  • Selección de variable secundaria: Medible, correlacionada con variable principal, y la perturbación más frecuente debe entrar entre las dos variables.

Control Feedforward — Directo de Referencia y Disturbio
// FEEDFORWARD DE DISTURBIO · DIRECTO DE REFERENCIA · Gff(s) · COMBINADO FB+FF
El control feedforward (o de prealimentación) es proactivo: actúa sobre una perturbación medible antes de que esta afecte la variable controlada, en lugar de esperar a que el error aparezca (como lo hace el feedback). Su efectividad depende de la calidad del modelo — si el modelo es exacto, rechaza completamente la perturbación. En la práctica se combina siempre con un controlador FB para manejar errores de modelo e incertidumbre.
Control Feedforward de Disturbio
📐 Derivación del Controlador FF Ideal
Objetivo: Que la perturbación D(s) no tenga efecto sobre la salida Y(s)
Modelo: Y(s) = G(s)·U(s) + Gd(s)·D(s)
Con control FF: U(s) = Gff(s)·D(s) + Gc(s)·E(s)
Para Y/D = 0 (rechazo perfecto):
G(s)·Gff(s)·D(s) + Gd(s)·D(s) = 0
G(s)·Gff(s) = −Gd(s)
Gff(s) = −Gd(s)/G(s) ← controlador FF ideal
Si G=Kp·e^(−θs)/(τs+1) y Gd=Kd·e^(−θds)/(τds+1):
Gff(s) = −(Kd/Kp)·(τs+1)/(τds+1)·e^(−(θd−θ)s)
FIG. 7 — Control combinado FB+FF. El FF compensa la perturbación D calculando MV_FF=-Gd/G·D. El FB corrige los errores residuales debidos a imprecisión del modelo. La suma MV_FF+MV_FB va al proceso.
Control Directo de Referencia (Feedforward de SP)
📐 Feedforward de Referencia
Objetivo: seguimiento perfecto de cambios de SP
Estructura: Gff_R(s) en paralelo con el lazo FB
Ideal: Gff_R(s) = G⁻¹(s) → U_FF = SP/G → Y=SP
Práctica: Gff_R(s) = Kp_inv·(τs+1)/1 (lead compensator)
U_total = Gc·E + Gff_R·R → respuesta al SP más rápida sin cambiar el rechazo de perturbaciones
IMPLEMENTACIÓN PRÁCTICA
Feedforward Estático vs Dinámico
Estático (Kff = −Kd/Kp): Solo corrige el efecto en estado estacionario. Fácil de implementar en DCS. Elimina el 70-80% del efecto de la perturbación.

Dinámico (Gff = −Gd/G): Incluye la dinámica completa — incluye lead/lag. Requiere conocer θd y τd. Elimina el 90-95% del efecto si el modelo es bueno.

FF de disturbio — Temperatura de alimentación en intercambiador:
Proceso: G(s)=2/(6s+1), con tiempo muerto θ=1 min. Disturbio: variación temperatura entrada, Gd(s)=0.5/(3s+1) con θd=0.5 min. Diseñar el feedforward estático y dinámico.

▸ SOLUCIÓN
FF estático: Kff = −Kd/Kp = −0.5/2 = −0.25
Si la temperatura de alimentación sube 10°C: u_FF = −0.25×10 = −2.5% en apertura de válvula de vapor
FF dinámico ideal:
Gff(s) = −Gd(s)/G(s) = −[0.5/(3s+1)]·e^(−0.5s) / [2/(6s+1)]·e^(−1s)]
= −(0.5/2)·(6s+1)/(3s+1)·e^(−(0.5−1)s)
= −0.25·(6s+1)/(3s+1)·e^(0.5s)
Problema: e^(+0.5s) → predicción futura → no realizable físicamente.
Solución práctica: Si θd < θ, usar: Gff(s) ≈ −0.25·(6s+1)/(3s+1) (sin el término de predicción)
FF estático: Kff=−0.25. FF dinámico: lead (6s+1), lag (3s+1), ganancia −0.25. El lead acelera la respuesta del FF anticipando el efecto del disturbio.

Diagramas de Bloques en Logica de Control
// ALGEBRA DE BLOQUES · REDUCCION · COMPARACION ELECTRICA · ACCIDENTES INDUSTRIALES
Los diagramas de bloques son la representacion grafica de sistemas de control en el dominio de Laplace. Cada bloque contiene una funcion de transferencia, las flechas indican el flujo de señales, y los circulos son sumadores o restadores. El algebra de bloques permite reducir sistemas complejos con lazos anidados a una sola funcion de transferencia equivalente.
Algebra de Bloques — Reglas Fundamentales

Ejemplo B1 — Reduccion de diagrama de bloques: Control Cascada
Sistema con lazo esclavo Gc2*G2 y lazo maestro Gc1*G1. Obtener T(s) = Y1(s)/R(s) y la funcion de transferencia de rechazo a perturbacion D2(s)/Y1(s).

REDUCCION PASO A PASO
PASO 1 — Reducir lazo esclavo (realimentacion unitaria):
LC2(s) = Gc2(s)*G2(s) / [1 + Gc2(s)*G2(s)] ← bloque equivalente del esclavo
PASO 2 — Sistema resultante: Gc1 en serie con LC2 en serie con G1, con realimentacion unitaria:
G_cadena(s) = Gc1(s) * LC2(s) * G1(s)
PASO 3 — Lazo cerrado maestro:
T(s) = G_cadena(s) / [1 + G_cadena(s)]
T(s) = [Gc1*LC2*G1] / [1 + Gc1*LC2*G1]
PASO 4 — Funcion de transferencia de perturbacion D2 (entra entre esclavo y G1):
Y1(s)/D2(s) = G1(s) * [1 - LC2(s)] / [1 + Gc1(s)*LC2(s)*G1(s)]
Si |Gc2*G2| >> 1 a bajas frecuencias: LC2 → 1, entonces Y1/D2 → 0. La perturbacion D2 es completamente rechazada por el lazo esclavo — clave del control cascada.
Comparacion con Sistemas Electricos — Analogias Fundamentales
Los sistemas electricos y los procesos quimicos obedecen las mismas ecuaciones diferenciales lineales de primer orden. Esta analogia no es coincidencia: ambos cumplen principios de conservacion (carga, masa, energia). Comprender la analogia permite usar la intuicion electrica para entender la dinamica de procesos y viceversa.

Accidentes Industriales por Falta de Control de Procesos
Advertencia academica: Los siguientes casos se presentan exclusivamente con fines educativos para ilustrar las consecuencias criticas de fallas en sistemas de control de procesos. El analisis de accidentes es una herramienta fundamental de la ingenieria de seguridad de procesos (PSM) y una obligacion profesional del ingeniero quimico.
BHOPAL, INDIA — 1984
Union Carbide — 3,500+ victimas
Fallo catastrofico del sistema de enfriamiento del tanque de MIC (isocianato de metilo). El sistema de scrubber y antorcha estaban desactivados por mantenimiento. La temperatura del tanque supero el punto de ebullicion del MIC por falta de control. Falla de control: sensores de temperatura y presion mal calibrados o ignorados. Leccion: sistemas criticos nunca deben operar sin instrumentacion funcional ni alarmado activo.
TEXAS CITY, USA — 2005
BP Refinery — 15 muertos, 180 heridos
Desbordamiento de la torre de destilacion de isomerizacion (ISOM). El nivel del reflujo se monitoreo con un indicador de nivel defectuoso que mostraba nivel bajo cuando en realidad era alto. El controlador de nivel siguio añadiendo alimentacion. Falla de control: indicador de nivel falso positivo + ausencia de alarma de alto nivel independiente. Leccion: redundancia de instrumentacion en variables criticas (SIL).
CHERNOBYL, UCRANIA — 1986
Reactor RBMK — Peor desastre nuclear
La prueba de seguridad redujo la potencia del reactor a niveles donde el coeficiente de vacio de moderacion se volvio positivo (inestabilidad intrinseca). Al insertar las barras de control de emergencia, un defecto de diseño produjo un pico de reactividad. Falla de control: sistema opero en zona intrinsecamente inestable + defecto en diseño de actuadores. Leccion: el diseño de sistemas de control debe garantizar estabilidad termodinamica intrinseca.
CHALLENGER, USA — 1986
NASA — 7 astronautas
Aunque no es un proceso quimico clasico, el fallo de los O-rings fue un fallo de control de temperatura: los sellos operaron fuera de su rango de temperatura validada (-2°C vs los 11°C de temperatura minima de lanzamiento valida). Falla de control: decision de lanzar fuera del envelope de diseño del proceso. Leccion: los limites de operacion validados deben ser respetados absolutamente (HAZOP, MOC).

Analisis Tecnico — ¿Que Falla de Control Causo Cada Accidente?
Bhopal 1984 — Analisis de la Cadena de Fallas de Control
+
Descripcion del sistema de control: El tanque de MIC debia mantenerse a T < 5°C con un sistema de refrigeracion por Freon. El gas liberado debia pasar por un scrubber de NaOH y una antorcha de gas.

Fallas de control identificadas:
1. El sistema de refrigeracion fue desactivado meses antes para ahorrar costos.
2. Los sensores de temperatura y presion del tanque no tenian alarmas independientes activas.
3. El sistema de alarmado DCS fue ignorado por los operadores por falsas alarmas previas (problema de alarm flooding, ISA-18.2).
4. El scrubber y la antorcha estaban en mantenimiento sin protocolo de gestion del cambio (MOC).

Desde la perspectiva del control: El lazo de temperatura del tanque operaba en lazo ABIERTO (sin accion correctiva activa). Cualquier perturbacion (agua que entro al tanque) no podia ser compensada. La respuesta dinamica fue una reaccion exotermica no controlada con T → 200°C en minutos.

Medidas preventivas modernas: SIL (Safety Integrity Level) IEC 61511, sistemas de instrumentacion de seguridad (SIS) independientes del DCS, redundancia de sensores (1oo2, 2oo3), prueba periodica de SIS, gestion de alarmas ISA-18.2.
Texas City 2005 — Fallo del Lazo de Nivel de Torre ISOM
+
El lazo de control involucrado: LIC (Level Indicating Controller) de la torre de isomerizacion. El instrumento de nivel era un flotador mecanico tipo bota (boot), susceptible a fallos de lectura. La torre se alimentaba con nafta para ser procesada.

Analisis del fallo del lazo:
- LI mostraba 8 inches (bajo) cuando en realidad el nivel era de 98 inches (lleno).
- El operador siguio añadiendo nafta porque el controlador de nivel pedia mas.
- La torre se desbordio y la nafta caliente (200°C+) cayo al suelo, vaporizandose.
- Un motor encendido causo la explosion e incendio.

Conclusion de control: Un solo sensor critico sin respaldo (1oo1) en un proceso con alto riesgo. La norma API 670 y la IEC 61511 requieren nivel de integridad SIL-2 para este tipo de variables criticas, implicando minimo redundancia 1oo2.

Implementacion moderna: Radar de nivel como sensor primario + flotador como respaldo + alta alta alarma de nivel independiente en SIS (Safety Instrumented System) separado del DCS.
Lecciones para el Ingeniero Quimico — Lista de Verificacion
+
Lista de verificacion de diseño de control seguro:

1. Toda variable critica tiene alarma y disparo (trip) independientes del lazo de control normal. Las alarmas del DCS son advertencias; el SIS es la ultima linea de defensa.

2. Nunca operar sin instrumentacion funcionando: Si un sensor critico falla o da lecturas dudosas, el proceso debe detenerse o pasar a modo seguro (fail-safe).

3. La redundancia de sensores es obligatoria para variables criticas: Nivel, temperatura y presion en puntos criticos deben tener minimo 2 sensores independientes (1oo2 o 2oo3).

4. La gestion del cambio (MOC) aplica a los sistemas de control: Ninguna modificacion de parametros, lazos o instrumentacion se hace sin analisis de riesgo previo.

5. Las alarmas deben ser significativas: Un sistema con muchas alarmas falsas lleva a que los operadores las ignoren. ALARM FLOODING = peligro. Norma ISA-18.2.

6. Las pruebas periodicas del SIS son obligatorias (Proof Test Interval, PTI). Un sistema de seguridad que no se prueba puede estar silenciosamente fallido.

7. El analisis HAZOP (Hazard and Operability Study) identifica desviaciones de parametros de proceso (presion, temperatura, flujo, nivel, composicion) y sus consecuencias. Es el puente entre el diseño de control y la seguridad del proceso.

Un sistema en lazo cerrado tiene G(s) = 5/(s+2) y H(s) = 1 (realimentacion unitaria). ¿Cual es la ganancia estatica T(0) del lazo cerrado?
A. T(0) = 5 (igual que lazo abierto)
B. T(0) = 2.5
C. T(0) = 5/7 = 0.714
D. T(0) = 1.0
Correcto. T(s) = G/(1+G*H) = [5/(s+2)] / [1 + 5/(s+2)] = 5 / (s+2+5) = 5/(s+7). Evaluando en s=0: T(0) = 5/7 = 0.714. El lazo cerrado reduce la ganancia de 5/2=2.5 a 5/7=0.714, pero tambien acelera el sistema (tau de 0.5s a 1/7s).
