Lazo Abierto vs Lazo Cerrado
// REALIMENTACIÓN NEGATIVA · GANANCIA · VELOCIDAD · ERROR · PERTURBACIONES
La diferencia fundamental entre lazo abierto y cerrado es la realimentación: en lazo cerrado, la salida del proceso se mide continuamente y se compara con el set-point para calcular el error y generar la acción correctiva. Esta realimentación negativa es lo que da al lazo cerrado sus propiedades superiores de rechazo de perturbaciones y reducción de error — pero también introduce el riesgo de inestabilidad.
LAZO ABIERTO
✗ Sin realimentación de la variable controlada
✗ Error de offset permanente si hay perturbaciones
✗ Sensible a cambios del proceso (Kp, τ, θ)
✓ Siempre estable (no puede volverse inestable)
✓ Simple: no requiere sintonización de Gc
✓ Útil para procesos muy conocidos y estables
Ejemplo: precalentador con válvula en posición fija
vs
LAZO CERRADO
✅ Realimentación negativa corrige error continuamente
✅ Elimina offset (con acción I) ante perturbaciones
✅ Robusto ante variaciones del proceso
✅ Rechaza perturbaciones de carga automáticamente
⚠ Puede volverse inestable si Gc mal sintonizado
⚠ Requiere transmisor, controlador y sintonización
Ejemplo: lazo TIC de temperatura en reactor exotérmico

Acciones de Control — P · I · D
// PROPORCIONAL · INTEGRAL · DERIVATIVA · PI · PD · PID IDEAL Y REAL
La ley de control PID combina tres acciones matemáticas sobre el error e(t)=SP−PV. Cada acción cumple una función específica: P corrige proporcionalmente al error presente, I elimina el error acumulado histórico y D anticipa el error futuro. Comprender el impacto individual de cada acción es esencial para sintonizar el controlador de forma racional.
📐 Ley de Control PID — Formas Paralela, Serie e Ideal
Forma paralela (estándar ISA):
u(t) = Kc·[e(t) + (1/τI)·∫e(t)dt + τD·de(t)/dt] + u_bias
Función de transferencia (forma paralela, dominio s):
Gc(s) = Kc·[1 + 1/(τI·s) + τD·s] = Kc·(τI·τD·s² + τI·s + 1)/(τI·s)
Forma serie (interactuante — DCS industriales):
Gc(s) = Kc·(1 + 1/(τI·s))·(1 + τD·s) ← introduce cero adicional
PID real (filtro derivativo, N=5–20 típico):
Gc(s) = Kc·[1 + 1/(τI·s) + τD·s/(τD·s/N + 1)] ← evita amplificación de ruido en D
Impacto de Cada Acción de Control
Acción Proporcional — P
Salida proporcional al error actual. Es la base de todo controlador. Aumentar Kc hace el lazo más rápido pero reduce los márgenes de estabilidad. P
u_P(t) = Kc·e(t) → Gc_P(s) = Kc
↑ Kc: velocidad ↑, amortiguamiento ↓, margen ↓
Offset permanente en SS ante perturbaciones
Acción Integral — I
Elimina el offset acumulando el error en el tiempo. Cuanto mayor el error acumulado, mayor la corrección. Peligro: windup ante saturación del actuador. I
u_I(t) = (Kc/τI)·∫e(t)dt → Gc_I(s) = Kc/(τI·s)
↓ τI: eliminación offset más rápida, ↑ oscilación
e_ss = 0 para entradas escalón (sistema tipo 1)
Acción Derivativa — D
Anticipa el error futuro usando su tasa de cambio. Agrega amortiguamiento → reduce sobreimpulso. Amplifica ruido de alta frecuencia → siempre usar con filtro N. D
u_D(t) = Kc·τD·de(t)/dt → Gc_D(s) = Kc·τD·s
↑ τD: amortiguamiento ↑, ruido amplificado ↑
No usar en procesos con ruido significativo o resp. inversa

Anti-windup en acción integral: Cuando la salida del controlador satura el actuador (válvula al 100% o 0%), la acción integral continúa acumulando error aunque la salida ya no pueda incrementarse. Al desaparecer la saturación, el integrador "desenrolla" produciendo un sobreimpulso severo. Los DCS industriales siempre implementan protección anti-windup: back-calculation, conditional integration o clamping. Sin anti-windup, el control PI/PID puede ser peligroso en arranques o cambios grandes de SP.

Especificaciones e Índices de Desempeño
// ISE · IAE · ITAE · ITSE · ESPECIFICACIONES TEMPORALES · CRITERIOS DE DISEÑO
Antes de sintonizar el controlador, el ingeniero debe definir qué tan bien debe comportarse el lazo. Las especificaciones temporales (OS%, ts, tr, tp) y los índices de desempeño (ISE, IAE, ITAE) proveen métricas cuantitativas que permiten comparar objetivamente diferentes sintonías y optimizar la respuesta de forma matemática.

Índices Integrales de Desempeño
ISE — Integral of Squared Error
Penaliza fuertemente los errores grandes (cuadráticos). Optimiza la sintonía más agresiva: respuesta rápida pero mayor sobreimpulso. Ideal cuando los errores grandes son inaceptables.
ISE = ∫₀^∞ e²(t) dt [mínimo → óptimo]
IAE — Integral of Absolute Error
Penaliza igual todos los errores. Balance entre rapidez y amortiguamiento. Más robusto que ISE. El criterio más usado en la industria química para sintonización práctica.
IAE = ∫₀^∞ |e(t)| dt [mínimo → óptimo]
ITAE — Integral Time Absolute Error
Penaliza más los errores tardíos que los iniciales (peso t). Produce la sintonía más amortiguada — mínimo sobreimpulso — a costa de respuesta inicial más lenta. Recomendado para procesos de seguridad.
ITAE = ∫₀^∞ t·|e(t)| dt [mínimo → óptimo]
ITSE — Integral Time Squared Error
Combina el peso temporal con penalización cuadrática. Balance entre ISE e ITAE. Menos usado que IAE/ITAE pero útil para procesos donde tanto la magnitud como la persistencia del error son críticas.
ITSE = ∫₀^∞ t·e²(t) dt [mínimo → óptimo]

Métodos Empíricos de Sintonización PID
// ZIEGLER-NICHOLS · COHEN-COON · IMC · LAMBDA TUNING · LAZO ABIERTO Y CERRADO
Los métodos empíricos sintetizan reglas de sintonización derivadas de experimentos en procesos FOPDT reales. No requieren modelos matemáticos exactos — solo los parámetros identificados experimentalmente: Kp, τ y θ (prueba de lazo abierto) o Kc_ult y P_ult (prueba de lazo cerrado de Ziegler-Nichols). Son el punto de partida obligado en la industria.
Ziegler-Nichols — Lazo Abierto (Curva de Reacción)
📐 Procedimiento Experimental
1. Con el lazo en manual, aplicar escalón de 5–10% en la salida del controlador (MV)
2. Registrar la respuesta de PV hasta estado estacionario → curva en S
3. Trazar la tangente en el punto de inflexión de la curva en S
4. Identificar: θ (tiempo muerto) = intersección tangente con línea inicial
5. Identificar: τ (constante de tiempo) = diferencia entre θ y punto donde tangente cruza SS
6. Kp = ΔPV_ss / ΔMV (ganancia estática del proceso)

Diseño PID por Asignación de Polos
// POLINOMIO DESEADO · IGUALACIÓN DE COEFICIENTES · ΖETA Y OMEGA_N DESEADOS
La asignación de polos (pole placement) es un método analítico que diseña el controlador PID de forma que los polos del lazo cerrado queden exactamente en las ubicaciones deseadas en el plano s. Si el ingeniero desea ζ=0.707 y ωn=2 rad/s, calcula Kc, τI, τD algebraicamente igualando el polinomio característico obtenido con el polinomio deseado. Garantiza las especificaciones exactas — a diferencia de los métodos empíricos que son aproximados.
📐 Procedimiento de Asignación de Polos para Proceso FOPDT-Aproximado
Proceso (1er orden sin tiempo muerto — aproximado): G(s) = Kp/(τs+1)
Controlador PID paralelo: Gc(s) = Kc·(1 + 1/(τI·s) + τD·s)
Polinomio característico del LC: 1 + Gc(s)·G(s) = 0
→ τ·τI·s³ + [τI + Kp·Kc·τD·τI]·s² + [1 + Kp·Kc·τI]·s + Kp·Kc = 0
Polinomio deseado (polos ζ, ωn + polo rápido en −p, p≥5ωn):
P_des(s) = (s² + 2ζωn·s + ωn²)·(s + p)
= s³ + (2ζωn+p)·s² + (ωn²+2ζωn·p)·s + ωn²·p

8·τI·s³ + [τI + 1.5·Kc·τD·τI]·s² + [1 + 1.5·Kc·τI]·s + 1.5·Kc = 0
Polinomio deseado:
P_des(s) = (s²+2×0.707×1×s+1²)·(s+8) = (s²+1.414s+1)·(s+8)
= s³ + 9.414s² + 12.312s + 8
Igualando coeficientes (dividiendo por coef. de s³ = 8/τI):
De s⁰: 1.5·Kc = 8·τI/τ × ... → 1.5·Kc/τI = 8/8 → Kc/τI = 1/1.5 = 0.667
De s⁰ igualado: Kp·Kc/τI = ωn²·p = 1×8 = 8 → 1.5·Kc/τI = 8 → Kc/τI = 5.33
→ Kc = 8·τ/(Kp·p) = 8×8/(1.5×8) = 64/12 = 5.33
De s² igualado: τI + 1.5·Kc·τD·τI = τ×9.414 = 8×9.414 = 75.31
De s¹ igualado: 1 + 1.5·Kc·τI = τ×12.312 = 98.5 → 1.5·5.33·τI = 97.5 → τI = 97.5/7.995 = 12.2 min
τD: 75.31 = 12.2 + 1.5×5.33×τD×12.2 → τD×97.55 = 63.11 → τD = 0.647 min
PID diseñado: Kc=5.33, τI=12.2 min, τD=0.647 min → polos en s=−0.707±j0.707 y s=−8 ✓
📐 Cómo elegir ζ, ωn y el polo rápido p para el diseño
+
Selección de ζ (coeficiente de amortiguamiento):
• ζ = 0.707 (1/√2): estándar industrial — OS≈4.3%, balance óptimo velocidad/amortiguamiento
• ζ = 0.5: permite 16% OS — más rápido, aceptable si OS no es crítico
• ζ = 1.0: sin sobreimpulso — para reactores exotérmicos o procesos de seguridad

Selección de ωn (frecuencia natural deseada):
• ωn = 1/τ: lazo cerrado igual de rápido que el proceso → buena partida
• ωn = 3/τ: lazo 3× más rápido — verificar que θ no limite (θ<1/ωn)
• Restricción: ωn ≤ 1/(3θ) para garantizar estabilidad con tiempo muerto

Selección del polo rápido p:
• p ≥ 5·ωn: polo no dominante — se "cancela" en la respuesta
• p = 10·ωn: muy rápido → parámetros PID más exigentes (τD muy pequeño)
• En la práctica: p = 5–8·ωn es el rango más usado

Limitación: Para procesos con tiempo muerto significativo (θ/τ>0.3), la aproximación de primer orden linealizado del tiempo muerto (1−θs) puede introducir error. En esos casos, usar IMC o verificar con Bode que Pm≥30°.

B. s = −√2 ± j√2 ≈ −1.414 ± j1.414

Controladores PID Digitales
// DISCRETIZACIÓN · TUSTIN (BILINEAL) · ZOH · MAPEO POLO-CERO · PERÍODO DE MUESTREO
En la práctica industrial, los controladores PID se implementan en microprocesadores, PLCs y DCS que operan en tiempo discreto: muestrean la señal de PV cada T segundos y calculan una nueva señal MV. El controlador continuo Gc(s) debe convertirse a su equivalente discreto Gc(z) mediante un método de discretización adecuado. La elección del período de muestreo T es crítica para preservar el comportamiento del controlador continuo.
Métodos de Discretización
Euler Hacia Adelante (Forward)
Aproximación de la derivada como diferencia hacia adelante. Simple pero puede ser inestable incluso para sistemas estables continuos. Rara vez usada en control.
s → (z−1)/T → integrador: 1/s → T/(z−1)
Euler Hacia Atrás (Backward)
Más estable que forward. Mapea el semiplano izquierdo de s en el interior del círculo unitario de z. Introduce pequeño error de fase. Usada en PLCs industriales simples.
s → (z−1)/(T·z) → integrador: T·z/(z−1)
Tustin / Bilineal (Recomendado)
Mapeo exacto del eje jω → círculo unitario. Preserva mejor la respuesta en frecuencia. Introduce distorsión de frecuencia ("warping") corregible con prewarping. Estándar industrial.
s → 2(z−1)/[T(z+1)] → bilineal exacto
Mapeo Polo-Cero (ZOH)
Mapea cada polo y cero de Gc(s) individualmente: pᵢ → e^(pᵢT) en el plano z. Preserva exactamente la respuesta al escalón. Útil para T grande relativo a τ del controlador.
pᵢ → e^(pᵢ·T) y zⱼ → e^(zⱼ·T) (mapeo exacto)
PID Discreto — Tustin (Forma Posicional y Velocidad)
📐 Algoritmo PID Discreto — Derivación con Tustin
Gc(s) = Kc·[1 + 1/(τI·s) + τD·s] → sustiuir s = 2(z−1)/[T(z+1)]
Término proporcional: uP[k] = Kc·e[k]
Término integral (Tustin): uI[k] = uI[k−1] + (Kc·T)/(2·τI)·(e[k] + e[k−1])
Término derivativo (Tustin): uD[k] = (2τD−T)/(2τD+T)·uD[k−1] + 2Kc·τD/(2τD+T)·(e[k]−e[k−1])
Salida total: u[k] = uP[k] + uI[k] + uD[k] + u_bias
Forma de velocidad: Δu[k] = u[k] − u[k−1] → evita reset windup automáticamente y facilita el paso manual↔automático

Arquitecturas de Control
// SERIE · PARALELO · ESTRUCTURAS AVANZADAS · 2-DOF · CONTROL RATIO
Las arquitecturas de control determinan cómo se conectan los bloques del lazo: el controlador, el proceso, la realimentación y las perturbaciones. El lazo PID simple (1 grado de libertad) es el punto de partida, pero muchos problemas industriales requieren estructuras más sofisticadas que desacoplen el seguimiento del SP del rechazo de perturbaciones, o que manejen procesos de gran tiempo muerto o con perturbaciones medibles.
Control Serie vs Paralelo
CONTROL SERIE (2-DOF)
Dos Grados de Libertad
El controlador tiene dos entradas: SP directamente y el error. El filtro de referencia Fr(s) modifica la respuesta al cambio de SP sin afectar el rechazo de perturbaciones. Permite optimizar seguimiento y perturbación de forma independiente.
U(s) = Gc(s)·[Fr(s)·R(s) − Y(s)]
Fr(s) = 1/(τf·s+1) típico
CONTROL PARALELO
Un Grado de Libertad
Estructura estándar: el controlador actúa sobre el error E=R−Y. Los parámetros Kc, τI, τD deben balancear simultáneamente el seguimiento de SP y el rechazo de perturbaciones — lo que a veces genera compromiso entre los dos objetivos.
U(s) = Gc(s)·E(s) = Gc(s)·[R(s) − Y(s)]
FIG. 4 — Comparativa 1-DOF vs 2-DOF. En 2-DOF, el filtro Fr(s) acondiciona el cambio de SP antes del sumador, permitiendo que Gc se optimice únicamente para rechazar perturbaciones. Fr(s)=1/(τf·s+1) con τf≈τD típicamente.
📐 Control de Relación (Ratio Control)
+
El control de relación mantiene la proporción entre dos caudales: F₂/F₁ = R_deseado (ratio). Común en: proporción aire/combustible en hornos, relación de alimentación en reactores, mezcla de corrientes.

Estructura:
1. F₁ se mide (variable "wild" — no controlada)
2. SP₂ = R × F₁ (set-point calculado en tiempo real para F₂)
3. Controlador de flujo FIC-2 mantiene F₂ = SP₂

Con corrección de ratio: Un analizador en línea mide la composición resultante → ajusta R en tiempo real → forma un lazo cascada con el control de ratio.

En MATLAB: Se implementa como dos lazos: un bloque multiplicador R×F₁ genera el SP del segundo lazo FIC-2.
❓ En una arquitectura 2-DOF, ¿qué objetivo cumple el filtro de referencia Fr(s)?
A. Aumenta la ganancia del controlador para respuestas más rápidas
B. Elimina el tiempo muerto del proceso
C. Acondiciona los cambios de SP para independizar el seguimiento del rechazo de perturbaciones
D. Reemplaza la acción integral del controlador PID
✅ Fr(s) modifica cómo el SP entra al lazo, sin cambiar el rechazo de perturbaciones. Permite sintonizar Gc agresivamente para perturbaciones y suavizar el seguimiento de SP con Fr(s)=1/(τf·s+1). Los dos objetivos son ahora independientes.
Predictor de Smith
// COMPENSACIÓN DE TIEMPO MUERTO · MODELO INTERNO · LAZO PREDICTOR · θ GRANDE
¿Cuándo usar el Predictor de Smith? Cuando θ/τ ≥ 0.3. Para procesos con gran tiempo muerto (columnas de destilación, hornos de crudización, procesos con transporte neumático largo), el PID convencional tiene márgenes de estabilidad muy reducidos — cualquier Kc razonable produce oscilación. El Predictor de Smith "saca" el tiempo muerto del lazo de control, permitiendo diseñar el PID como si el proceso no tuviera θ.
Principio del Predictor de Smith
📐 Estructura del Predictor
Proceso real: G(s) = Gm(s)·e^(−θs) donde Gm=Kp/(τs+1)
Modelo interno: Ĝ(s) = Ĝm(s)·e^(−θ̂s) (estimación del proceso)
Señal predictor: yp(s) = Gc·Gm·(1−e^(−θs))·Y_sinθ(s)
Efecto: El lazo de control ve el proceso SIN tiempo muerto: Gm(s)
El PID se diseña para Gm(s) = Kp/(τs+1) → puede usar Z-N, IMC o asignación de polos normales
Función de transferencia resultante (modelo exacto):
T(s) = Gc·Gm·e^(−θs) / [1 + Gc·Gm] ← el e^(−θs) solo aparece en el numerador — no en el denominador
FIG. 5 — Diagrama del Predictor de Smith. El predictor genera una estimación de la salida sin tiempo muerto a partir del modelo interno Ĝm y la elimina de la realimentación al controlador. El PID solo ve Gm(s) — diseño trivial.

Predictor de Smith — Columna de destilación Manuelita:
G(s) = 0.8·e^(−12s)/(25s+1). θ/τ = 0.48 → control PID convencional muy limitado. Diseñar el predictor de Smith con PID IMC (λ=10 min) para el proceso sin tiempo muerto Gm(s)=0.8/(25s+1).

▸ SOLUCIÓN
Sin predictor (PID Z-N para referencia):
Kc_ZN = 1.2×25/(0.8×12) = 30/9.6 = 3.125. Pero Pm será solo ~18° — lazo muy agresivo.
Con Predictor de Smith — diseñar PID para Gm(s) sin θ:
IMC-PI para Gm(s): Kc = τ/[Kp·(λ)] = 25/[0.8×10] = 3.125 (¡mismo Kc!)
τI = τ = 25 min (no hay θ/2 porque usamos Gm sin θ)
Diferencia fundamental:
Sin predictor: polos LC en zona inestable para Kc=3.125 con θ=12
Con predictor: polos LC de Gm(s) bien ubicados — e^(−θs) solo en numerador
En MATLAB:
Gm = tf(0.8,[25 1]); Gc = pid(3.125, 3.125/25);
Smith_predictor = feedback(Gc*Gm,1)*tf(1,1,'InputDelay',12);
El predictor de Smith permite Kc=3.125 con respuesta estable. Sin predictor, Kc máximo ≈ 0.5 para Pm≥30°.
Limitaciones del Predictor de Smith: (1) Es sensible a errores en el modelo — si el θ real difiere del modelo en más del 20%, el predictor puede desestabilizarse. (2) No funciona bien con procesos integradores (nivel sin válvula de salida). (3) Para perturbaciones de carga, el predictor no ofrece ventaja sobre el PID si la perturbación entra después del tiempo muerto. Alternativa moderna: Control Predictivo Basado en Modelo (MPC) — maneja tiempo muerto, restricciones y múltiples variables simultáneamente.
Control Cascada
// LAZO MAESTRO · LAZO ESCLAVO · RECHAZO ANTICIPADO · θ/τ EFECTIVO
El control cascada usa dos controladores en serie: el lazo esclavo (interno, más rápido) controla una variable secundaria, y el lazo maestro (externo, más lento) controla la variable principal y genera el set-point del esclavo. La clave: el esclavo rechaza las perturbaciones internas antes de que afecten la variable principal. Resultado: el lazo maestro "ve" un proceso más rápido y lineal.
FIG. 6 — Control cascada. La perturbación D₂ interna es rechazada por el lazo esclavo antes de propagarse a la variable principal Y₁. El lazo maestro solo ve G₁(s)·LC₂(s) — un proceso más rápido y lineal.
Reglas de Diseño para Control Cascada
  • Regla de velocidad: El lazo esclavo debe ser al menos 3–5× más rápido que el lazo maestro. Si τ₂ > τ₁/3, la cascada no aporta beneficio sobre el lazo simple.
  • Sintonizar primero el esclavo: Con el lazo maestro en manual, sintonizar el esclavo (Gc₂) hasta que responda rápido y sin oscilación. Típicamente P o PI con ganancia alta.
  • Sintonizar el maestro: Con el esclavo en automático, el maestro ve G₁(s)·LC₂(s) como su proceso efectivo. Usar Z-N, IMC o CSD sobre este modelo combinado.
  • Selección de la variable secundaria: Debe ser medible, correlacionada con la variable principal, y la perturbación más frecuente debe entrar entre las dos variables (antes de G₁ pero después de G₂).

Cascada Temperatura — Intercambiador Incauca (Flujo de Vapor):
Proceso: Temperatura jugo G₁(s)=1.5/(8s+1). Variable secundaria: flujo de vapor G₂(s)=0.8/(0.5s+1). Perturbación de presión de vapor entra en G₂. La τ₂=0.5 ≪ τ₁=8 → cascada ideal. Diseñar Gc₂ (P) y Gc₁ (PI-IMC).

▸ SOLUCIÓN
Lazo esclavo Gc₂ (flujo, muy rápido, P puro):
Kc₂_ZN = τ₂/(Kp₂·θ₂) para θ₂≈0.05 min (pequeño) → Kc₂ = 0.5/(0.8×0.05) = 12.5 %/(m³/h)
LC₂(s) = feedback(Gc₂·G₂,1) = (12.5×0.8/0.5) / (s + 12.5×0.8/0.5 + 1/0.5) = 20/(s+22) ≈ 0.909/(0.045s+1)
Proceso efectivo del maestro:
G_eff(s) = G₁(s)·LC₂(s) ≈ 1.5/(8s+1) × 0.909/(0.045s+1) ≈ 1.364/(8s+1) (esclavo muy rápido → ≈ ganancia)
Lazo maestro Gc₁ (PI-IMC, λ=5 min):
Kc₁ = τ₁/(Kp_eff·λ) = 8/(1.364×5) = 1.17
τI₁ = τ₁ = 8 min
Cascada: rechaza variaciones de presión de vapor 10× más rápido que lazo simple. SP de flujo cambia en ~1 s; temperatura en ~15 min sin cascada → ~5 min con cascada.
▸ CASCADA EN INDUSTRIA — REFINERÍA ECOPETROL BARRANCABERMEJA
En la torre de destilación atmosférica, la temperatura del plato 10 (TIC-10) es la variable principal. La variable secundaria es el flujo de reflujo (FIC-20). Perturbaciones frecuentes: variaciones de presión en la línea de reflujo, cambios de temperatura del reflujo. El lazo de flujo FIC-20 tiene τ₂≈5 s; el lazo de temperatura TIC-10 tiene τ₁≈15 min. Relación τ₁/τ₂ ≈ 180 → cascada muy efectiva. El lazo de flujo rechaza el 95% de las perturbaciones de presión antes de que afecten la temperatura. En operación: Kc₂=8, τI₂=0.1 min (PI rápido de flujo); Kc₁=1.5, τI₁=12 min (PI lento de temperatura).
Control Feedforward — Directo de Referencia y Disturbio
// FEEDFORWARD DE DISTURBIO · DIRECTO DE REFERENCIA · Gff(s) · COMBINADO FB+FF
El control feedforward (o de prealimentación) es proactivo: actúa sobre una perturbación medible antes de que esta afecte la variable controlada, en lugar de esperar a que el error aparezca (como lo hace el feedback). Su efectividad depende de la calidad del modelo — si el modelo es exacto, rechaza completamente la perturbación. En la práctica se combina siempre con un controlador FB para manejar errores de modelo e incertidumbre.
Control Feedforward de Disturbio
📐 Derivación del Controlador FF Ideal
Objetivo: Que la perturbación D(s) no tenga efecto sobre la salida Y(s)
Modelo: Y(s) = G(s)·U(s) + Gd(s)·D(s)
Con control FF: U(s) = Gff(s)·D(s) + Gc(s)·E(s)
Para Y/D = 0 (rechazo perfecto):
G(s)·Gff(s)·D(s) + Gd(s)·D(s) = 0
G(s)·Gff(s) = −Gd(s)
Gff(s) = −Gd(s)/G(s) ← controlador FF ideal
Si G=Kp·e^(−θs)/(τs+1) y Gd=Kd·e^(−θds)/(τds+1):
Gff(s) = −(Kd/Kp)·(τs+1)/(τds+1)·e^(−(θd−θ)s)
FIG. 7 — Control combinado FB+FF. El FF compensa la perturbación D calculando MV_FF=-Gd/G·D. El FB corrige los errores residuales debidos a imprecisión del modelo. La suma MV_FF+MV_FB va al proceso.
Control Directo de Referencia (Feedforward de SP)
📐 Feedforward de Referencia
Objetivo: seguimiento perfecto de cambios de SP
Estructura: Gff_R(s) en paralelo con el lazo FB
Ideal: Gff_R(s) = G⁻¹(s) → U_FF = SP/G → Y=SP
Práctica: Gff_R(s) = Kp_inv·(τs+1)/1 (lead compensator)
U_total = Gc·E + Gff_R·R → respuesta al SP más rápida sin cambiar el rechazo de perturbaciones
IMPLEMENTACIÓN PRÁCTICA
Feedforward Estático vs Dinámico
Estático (Kff = −Kd/Kp): Solo corrige el efecto en estado estacionario. Fácil de implementar en DCS. Elimina el 70-80% del efecto de la perturbación.

Dinámico (Gff = −Gd/G): Incluye la dinámica completa — incluye lead/lag. Requiere conocer θd y τd. Elimina el 90-95% del efecto si el modelo es bueno.

FF de disturbio — Temperatura de alimentación en intercambiador:
Proceso: G(s)=2/(6s+1), con tiempo muerto θ=1 min. Disturbio: variación temperatura entrada, Gd(s)=0.5/(3s+1) con θd=0.5 min. Diseñar el feedforward estático y dinámico.

▸ SOLUCIÓN
FF estático: Kff = −Kd/Kp = −0.5/2 = −0.25
Si la temperatura de alimentación sube 10°C: u_FF = −0.25×10 = −2.5% en apertura de válvula de vapor
FF dinámico ideal:
Gff(s) = −Gd(s)/G(s) = −[0.5/(3s+1)]·e^(−0.5s) / [2/(6s+1)]·e^(−1s)]
= −(0.5/2)·(6s+1)/(3s+1)·e^(−(0.5−1)s)
= −0.25·(6s+1)/(3s+1)·e^(0.5s)
Problema: e^(+0.5s) → predicción futura → no realizable físicamente.
Solución práctica: Si θd < θ, usar: Gff(s) ≈ −0.25·(6s+1)/(3s+1) (sin el término de predicción)
FF estático: Kff=−0.25. FF dinámico: lead (6s+1), lag (3s+1), ganancia −0.25. El lead acelera la respuesta del FF anticipando el efecto del disturbio.
▸ FEEDFORWARD INDUSTRIAL — HORNO DE CRUDIZACIÓN ECOPETROL CARTAGENA
El horno H-101 calienta crudo de 80°C a 350°C. La principal perturbación es la temperatura del crudo de entrada (varía ±15°C con los cambios de tanque de origen). Esta temperatura se mide con TT-5 aguas arriba. El controlador FF estático calcula: ΔFuego_FF = −(Cp·F·ΔT_entrada)/(Qmax_quemadores). En el DCS Honeywell Experion, el bloque LEADLAG implementa el FF dinámico: τ_lead=8 min (dinámica del crudo en horno), τ_lag=2 min (dinámica del lazo de combustión). Resultado: las variaciones de temperatura de entrada producen solo ±2°C en la salida (vs ±25°C sin FF).
❓ El feedforward ideal es Gff(s) = −Gd(s)/G(s). Si θd > θ (tiempo muerto del disturbio mayor que el del proceso), ¿qué ocurre?
A. El feedforward no puede implementarse por ser inestable
B. El feedforward es realizable: incluye e^(−(θd−θ)s) con exponente negativo (retardo puro)
C. El feedforward genera un polo en el semiplano derecho
D. No hay diferencia con el caso θd < θ
✅ Si θd > θ: Gff incluye e^(−(θd−θ)s) con exponente negativo → retardo puro → perfectamente realizable. Si θd < θ: e^(+(θ−θd)s) → predicción → no realizable físicamente → se aproxima sin el término de adelanto.

Aspen Hysys Dinámico — Flujo de Trabajo
  • Construir el modelo en modo estacionario: Definir corrientes, equipos (intercambiadores, columnas, reactores), propiedades termodinámicas (Peng-Robinson, NRTL). Converger el estado estacionario como punto de operación base.
  • Convertir a modo dinámico (Dynamic Mode): Hysys → Tools → Integrator. Especificar volúmenes de equipos, válvulas de control (Cv, acción AO/AC), transmisores (rangos y time constants).
  • Agregar los controladores PID: Insertar bloques PID desde el Toolbox. Conectar PV (señal del transmisor), SP (valor deseado), OP (señal a válvula). Configurar modo: Auto/Manual, límites, forma de la ecuación PID.
  • Sintonizar en Hysys: Usar el autotuner interno (Tools → Controller Faceplate → Tuning → Autotuner → IMC). Introduce un paso en la variable manipulada y calcula Kc, τI, τD automáticamente.
  • Validar perturbaciones: Introducir perturbaciones vía Event Scheduler: cambio de temperatura de entrada, variación de flujo de alimentación, cambio de composición. Observar la respuesta del lazo y ajustar si necesario.
  • Exportar resultados: Grabar tendencias en Databook → exportar a Excel. Verificar que OS%, ts y e_ss cumplen las especificaciones del proyecto final 740021C.
